# Contributing to ME7540

Thank you for contributing to ME7540! We welcome improvements, bug fixes, and new features. To keep the codebase clean, maintainable, and consistent, please follow the guidelines below.

---

## 1. Code Style

- We follow **PEP8** coding standards.  
- All code must be **clean, readable, and concise**. Avoid large rewrites; focus on the task or issue at hand.  
- Use `ruff` to lint and format your code. Check for linting issues with `ruff check .` and automatically format with `ruff format .`  
- Type annotations are required where possible. Use `mypy .` to check your types.

---

## 2. Branches and Pull Requests

- Assign yourself to an issue before starting work.  
- **Open a pull request early**, even if your solution is incomplete. This allows reviewers to provide feedback and guide your implementation.  
- Keep PRs **focused and small** to simplify review and reduce merge conflicts.  
- Each PR must reference the issue it resolves, e.g., `Closes #123`.

---

## 3. Testing

- Test your code thoroughly before submitting a PR.  
- Include new tests if you are adding functionality or fixing a bug.  
- Ensure existing tests continue to pass.

---

## 4. Documentation and Citing Work

- Update any relevant documentation when adding or changing functionality.  
- Properly cite sources, especially AI-generated code. For example, `# Code generated with ChatGPT, adapted for ME7540`.  
- Include docstrings for all public functions, classes, and modules.

---

## 5. General Guidelines

- Communicate clearly on the GitHub issue or PR if you need clarification.  
- Strive for concise, maintainable solutions—avoid unnecessary complexity.  
- Ensure your code integrates smoothly with the existing codebase.

---

Following these guidelines will help keep ME7540 clean, consistent, and easy to maintain for everyone. Thank you for contributing!
